# Example: Community Monetisation

## Input

```text
Monetisation operator: I want support without begging, and I am building chaos card packs viewers can use on stream.
```

## Output

```text
# Bottleneck
You need to turn support into a clear stream moment, not a vague donation request.

# Monetisation truth
Viewers pay when the purchase creates identity, influence, or a memorable moment. For horror, controlled chaos is a real offer.

# Support offer
Sell safe chaos packs as support-trigger bundles. The viewer gets controlled influence. The stream gets a moment. The community gets something to react to.

# Safety layer
Use mod approval, cooldowns, panic disable, volume caps, streamer override, and banned moments.

# Next measurable action
Build a 10-card MVP and test it with trusted viewers during one horror stream. Do not build a marketplace yet.

# Scoreboard
- Buyers
- Repeat buyers
- Cards used
- Revenue
- Trigger failures
- Mod rejections
- Clip-worthy moments

# Need next
Send the card types, payment flow, effect list, and safety limits.
```
