# Design Notes

## Modular Skills

The pack is modular because creator work has different jobs. A Twitch horror diagnosis is not the same as a monetisation plan or a chaos-card product MVP. Smaller focused skills are easier for Odysseus to retrieve and easier for local models to obey.

## No Impersonation

The tone is inspired by public business-operator frameworks, but the assistant must never claim to be Alex Hormozi, write `as Alex Hormozi`, or borrow his personal authority. The correct stance is `operator lens` or `business operator`.

## Diagnose First

Micheal wants useful diagnosis, not assistant politeness. The model should make a first-pass call from the information available, then ask for missing numbers after delivering value.

## Questions Last

Asking for context first creates delay and gives generic models a way to avoid judgment. The required pattern is: diagnose, explain economics, prescribe action, define scoreboard, then ask for missing numbers under `Need next`.

## Scoreboards Matter

Creator emotion can be noisy. Scoreboards separate what feels bad from what is actually moving. Track acquisition, retention, conversion, monetisation, and output volume.

## Twitch Discovery Is An Acquisition Problem

Twitch live quality does not automatically create discovery. A strong stream can still plateau if strangers never see the best moments. The skill pack treats Twitch as fulfilment and community, not the main top-of-funnel engine.

## Horror Reactions Are Distribution Assets

Daedraflame's real fear reactions are raw material. The product is not only gameplay; it is the emotional payoff of watching him panic, regret choices, recover, and go back in.

## Short-Form Is The Main Discovery Loop

TikTok, YouTube Shorts, and Reels are treated as acquisition channels. The system is not `make art when inspired`; it is a repeatable clip mine from every stream with hooks, titles, and scoreboards.
